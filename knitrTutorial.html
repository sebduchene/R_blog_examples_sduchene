Tools for reproducible research: A tutorial on html report and markup files

By Sebastian Duchene at Aentropico Engineering

In this post we will show how to write markup files for R code and produce html reports with figures, code and comments embedded. This is a useful tool for web development and keeping track of data analyses.

When markup files and html reports are combined with cloud-based version control, such as GitHub, overall collaboration is much easier. In a future post we will look at using version control systems and repositories.

An introduction to markup

A markup language is used to annotate documents. It can be useful when we want to convey more information than we can with text only. A data analyst may want to annotate some code, so that the output of the analysis is included in some sort of report. To do this, simple code would need to be executed, and then the output included somehow in the code. Markup languages are ideal for this purpose, and may be more easy to follow than pasting code and output into a word document.

We can choose many different formats to generate reports for code, see some examples here. In this post we will consider html, but this can easily be extended to pdf or LaTeX formats.

In this post we will outline the basic syntax and walk though gernerating a report from end to end.

The R tools

We will be using the packages “knitr” and “markdown” by Yihui Xie and Jeffrey Horner, respectively. The websites for these packages contain plenty of information for more advanced report generation:

  • knitr
  • markdown

The first step will be to download and install the packages:

install.packages("knitr")
install.packages("markdown")

library(knitr)
library(markdown)

General syntax for markdown and knitr

Annotating R code for knitr requires some special syntax, we will explain some basics bellow:

  • Bold text and italics

For italics use a single asterix (*) around the text

*This is in italics*

to obtain:

This is in italics

For bold text use double asterix (**)

**This is some bold text**

to obtain:

This is some bold text

  • For H1 and H2 headers use (=) and (-) under the text:
This is a header of type H1
===========================

This is a header of type H2 
---------------------------

Which look like this:

This is a header of type H1

This is a header of type H2

A wider range of headers can be defined using hash (#) (up to six):

# This is header of type H1
## This is header of type H2
### This is header of type H3
#### This is header of type H4
##### This is header of type H5
###### This is header of type H6
  • For lists
- item one
- item two
- item three

Which looks like this:

  • item one
  • item two
  • item three

The lists can be ordered with numbers:

1. item one
2. item two
3. item three

Which looks like this:

  1. item one
  2. item two
  3. item three

URLs can be included:

[this is a website](http://www.aentropi.co/)

They look like ordinary clickable elements in the resulting html:

this is a website

Sytax for R code embedding and execution

So far we have seen what the syntax for some formatting. A much more extensive overview can be found here, or here.

What is most interesting about knitr is that chunks of code can be executed in R, and the output is printed to the markup.

For executable code, open a line with three back ticks and an {r} as follows (just delete the hash (#) in the box bellow). The {r} tells knitr to execute the code in R.

#```{r}
The code goes here. Delete the hash for execution.
#```

When any lines of code should be included but not actually executed, the {r} can be deleted:

#```
The code goes here. Remember to delete the hash.
#```

Any of the code chunks can include plots. These will be printed in separate graphics device for each of the code chunks.

Writing an example markup file in R

Now we can proceed to writing a markup file that uses the information described above.

  • Preparing a text file with code

We will first use some example code which is a y = x line with some random noise. Open a text file, copy the text bellow, and save the file with the name “my_markup.txt”

My markup file
==============

This is some annotation text to describe the code bellow.
This is a simulation of an y = x line with some added random normal noise

#```
set.seed(1234)
x <- runif(1000)
y <- x + rnorm(1000, 10)

print(data.frame(mean.x = mean(x), mean.y = mean(y)))

plot(x, y, pch = 20)
#```

The last code was not executed. In the following chunk we will execute the code in R and the output and plot will be embedded

#```{r}
set.seed(1234)
x <- runif(1000)
y <- x + rnorm(1000, 10)

print(data.frame(mean.x = mean(x), mean.y = mean(y)))

plot(x, y, pch = 20)
#```

Now delete the hash (#) tags before the back ticks and save the file (recall we are using the .txt extension in this example).

Using knitr to convert the file to Rnw format

Now open the R console and set the working directory to where you saved “my_markup.txt”. Assuming you have installed knitr and markdown, run the following code

library(knitr)
knit("my_markup.txt", "my_markup.rnw")

## 
## 
## processing file: my_markup.txt

## 
  |                                                                       
  |                                                                 |   0%
  |                                                                       
  |......................                                           |  33%
##   ordinary text without R code
## 
## 
  |                                                                       
  |...........................................                      |  67%
## label: unnamed-chunk-4

## 
  |                                                                       
  |.................................................................| 100%
##   ordinary text without R code

## output file: my_markup.rnw

## [1] "my_markup.rnw"

The file output of the function “knit” is a raw markup called “my_markup.rnw”, this file contains some of the output from the code we executed. Also note that there is a new folder in the working directory called “figure”. This contains the plot that we generated with the code. Do not move this folder because the figure will be used to produce the html.

Using markdown to convert the Rnw file and embed the figure into an html

At this point we will use the package markdown to convert the rnw file and the figure to a single html that we can open in any browser, or upload to a website.

The process is very brief. Type the following at the prompt:

markdownToHTML("my_markup.rnw", "my_markup.html")

Finally, you will find “my_markup.html” in the working directory. Open the file in any web browser to see the output.

Final remarks

Writing markup files can be straight forward. By now you can fiddle with the syntax to print better plots, as long as the same graphics device is used (ggplot2 or lattice work well). The key is to understand the syntax.
